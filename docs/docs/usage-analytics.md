# 使用分析

> 可视化多站点/账号的使用趋势，帮助分析成本和用量，支持查看请求数、Token 消耗、配额、延迟等指标。

## 功能概述

- **用量趋势图表**：查看每日/每周/每月的请求数、Token 消耗、配额使用情况
- **模型分布分析**：查看各模型的请求占比、Token 消耗占比、成本占比
- **账号对比**：对比不同账号/站点的使用量
- **使用时间热力图**：可视化每天各时段的使用量分布
- **延迟分析**：查看延迟趋势和慢请求分布
- **数据导出**：支持导出用量数据为 JSON 格式

## 前置条件

使用此功能前，需要先启用用量历史同步：

1. 进入 **设置 → 账号用量**
2. 开启 **「用量历史同步」** 开关
3. 配置同步参数：
   - **保留天数**：数据保留天数（建议 30-90 天）
   - **同步方式**：手动同步 / 自动同步
   - **自动同步间隔**：自动同步的频率
4. 点击 **「立即同步」** 获取初始数据

## 主要指标

| 指标 | 说明 |
|------|------|
| 请求数 (Requests) | API 调用总次数 |
| Prompt Tokens | 输入模型的总 Token 数 |
| Completion Tokens | 模型输出的总 Token 数 |
| Total Tokens | 总 Token 数 (Prompt + Completion) |
| 配额消耗 (Quota) | 消耗的配额金额 |
| 成本 (Cost) | 估算的费用（基于汇率） |

## 图表类型

### 1. 日概览趋势图

展示选定时间范围内的每日用量汇总，包括请求数、Token 总量、配额消耗。

### 2. 模型分布图

饼图/柱状图展示各模型的：
- 请求数分布
- Token 消耗分布
- 成本分布

### 3. 账号对比图

对比不同账号/站点的使用量，找出用量最大的账号。

### 4. 使用时间热力图

展示一周内各时段的用量分布，识别使用高峰时段。

### 5. 延迟趋势图

- 延迟趋势：展示平均延迟随时间的变化
- 延迟直方图：展示延迟分布
- 慢请求模型：列出响应最慢的模型

## 筛选功能

支持多维度筛选：
- **站点筛选**：选择要查看的站点
- **账号筛选**：选择要查看的账号
- **时间范围**：选择日期范围
- **Token 筛选**：选择特定的 API Key

## 导出功能

点击「导出」按钮可导出当前筛选条件下的用量数据为 JSON 文件，包含：
- 每日汇总数据
- 按模型聚合的数据
- 按账号聚合的数据

## 配置选项

| 选项 | 说明 | 默认值 |
|------|------|--------|
| 启用用量历史同步 | 开关 | 关闭 |
| 保留天数 | 数据保留天数 | 30 天 |
| 同步方式 | 手动/自动 | 手动 |
| 自动同步间隔 | 自动同步频率 | 6 小时 |
| 最小同步间隔 | 两次同步的最小间隔 | 30 分钟 |

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 无数据 | 先在「设置 → 账号用量」启用同步并点击「立即同步」 |
| 数据不完整 | 检查同步状态，确保各账号同步成功 |
| 图表加载慢 | 减少时间范围或筛选条件 |
