网站公告

自动抓取已添加账号所属站点的公告，把新公告集中保存在插件里，并按需通过任务通知提醒你。

适合什么场景

很多中转站会通过站点公告说明维护计划、模型变更、价格调整、充值活动或使用限制。如果你管理了多个账号，逐个打开站点查看公告很容易遗漏。

网站公告功能会在后台定时检查已启用账号关联的站点，并把新公告汇总到 网站公告 页面。你可以在那里筛选站点、查看公告内容、标记已读，也可以手动立即检查一次。

功能关系

网站公告本身负责 抓取、保存和展示公告；任务通知只负责在发现新公告后，把提醒发送到浏览器系统通知或第三方渠道。

因此这两个开关是分开的：

  • 启用网站公告抓取：控制是否后台定时检查站点公告。
  • 网站公告通知：控制新公告是否通过任务通知渠道发送提醒。

如果你只想集中查看公告，不需要弹出提醒，可以开启抓取、关闭通知。如果你关闭抓取，通知渠道也不会主动去站点读取公告。

入口位置

查看公告

  1. 打开 All API Hub 设置页。
  2. 在左侧菜单进入 网站公告。
  3. 查看已抓取的公告记录，或点击 立即检查 手动刷新。

开启或关闭后台抓取

  1. 打开 基础设置 → 通用。
  2. 找到 网站公告。
  3. 使用 启用网站公告抓取 开关控制后台定时检查。

关闭后台抓取后，已保存的本地公告记录不会被删除，网站公告 页面仍然可以手动检查。

配置新公告提醒

网站公告提醒使用 任务通知 的通知渠道。

  1. 打开 基础设置 → 通用 → 通知。
  2. 先开启 启用任务通知，并配置至少一个通知渠道。
  3. 在通知事件中开启 网站公告通知。

如果只关闭 网站公告通知，新公告仍会保存到本地记录，但不会发送系统通知或第三方通知。

抓取规则

项目说明
检查对象已添加且未禁用的账号。
默认间隔默认每 360 分钟检查一次。
间隔范围配置值会限制在 15 分钟到 24 小时之间。
去重方式同一站点的新公告会按内容或上游公告 ID 去重，避免重复记录。
本地保留每个站点最多保留 10 条公告记录。
手动检查网站公告 页面里的 立即检查 不受后台抓取开关影响。

对于共用同一个站点公告接口的账号，插件会按站点去重检查，避免同一站点因为多个账号被重复抓取。Sub2API 的公告是账号级未读公告，因此会按账号检查。

支持范围

网站公告目前主要支持两类来源：

来源抓取方式说明
One API / New API 兼容站点及其常见变体读取站点的 /api/notice 公告接口适用于多数 One API、New API、Veloera、OneHub、DoneHub 等兼容部署。具体是否可用取决于目标站点是否保留该接口并返回可用公告内容。
Sub2API读取账号的未读公告接口抓取未读公告后，会在成功通知或本地阅读时尽量同步上游已读状态。

如果某个站点没有可用公告接口，或接口返回为空，插件会把它视为暂时没有可展示的公告。你可以在 网站公告 页面按站点查看最近一次检查状态。

页面功能

网站公告 页面提供以下操作：

  • 汇总指标：查看公告总数、未读公告数和涉及站点数。
  • 筛选公告：按站点、站点类型和阅读状态筛选。
  • 展开内容：展开单条公告查看完整内容。
  • 标记已读：将单条公告或当前筛选范围内的公告标记为已读。
  • 立即检查：手动触发一次公告抓取，并刷新本地记录。
  • 查看来源：跳转到对应站点的公告或首页位置，便于打开原站确认详情。

常见问题

为什么没有任何公告记录？

可能原因包括：

  • 当前账号所属站点暂时没有公告。
  • 站点没有开放兼容的公告接口。
  • 账号被禁用，因此不会参与后台检查。
  • 浏览器后台任务尚未运行，可以进入 网站公告 页面点击 立即检查。

为什么显示“暂不支持公告接口”？

这表示最近一次检查没有发现可用公告接口，或目标站点的接口行为与插件当前适配的格式不一致。常见于私有改版、关闭公告接口、接口返回结构变更的站点。

为什么关闭通知后还能看到新公告？

网站公告通知 只控制是否把新公告发送到浏览器系统通知或第三方通知渠道。只要 启用网站公告抓取 仍然开启，新公告仍会保存到本地公告记录。

为什么关闭后台抓取后还能手动检查？

后台抓取开关只控制定时任务。手动检查是用户主动操作，用于临时刷新或排查站点状态，因此不会被该开关拦截。

Sub2API 的已读状态会同步到站点吗？

会尽量同步。插件在成功通知新公告后，会尝试把对应 Sub2API 公告标记为已读；你在本地展开未读 Sub2API 公告时，也会尝试同步上游已读状态。如果站点接口失败，本地已读状态仍会按当前操作更新。
